# 0.4.0 RC1 and Documentation Site Tracking Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Create the `bd` tracker structure for the `0.4.0-rc.1` prerelease and the documentation-site work required before the final `0.4.0` release.

**Architecture:** Keep prerelease work and documentation-site delivery separate in the tracker. Represent RC1 as a single release-prep task, represent the documentation site as an epic, and decompose that epic into concrete child tasks using `discovered-from` dependencies. Use `ascribe` as the implementation reference for a Scaladoc-based site (`docs/_docs`, `docs/_blog`, `docs/_assets`, `docs/sidebar.yml`) and record the intent in the checked-in changelog.

**Tech Stack:** `bd`, `CHANGELOG.md`, repository planning docs under `docs/plans`
